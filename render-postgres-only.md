## Full Setup: Render DB + Render Backend + Vercel Frontend

## Stack (Updated — No Supabase)

| Layer | Service | Free Tier |
|-------|---------|-----------|
| Frontend | Vercel | Free, unlimited deploys |
| Backend (API) | Render Web Service | Free, 750 hrs/month (sleeps) |
| Database | Render PostgreSQL | Free, 1GB storage |
| File Storage | Cloudflare R2 | 10GB free |
| Email | Resend | 3,000/month free |
| Auth | Custom JWT (backend) | No third-party needed |
| Realtime | Socket.IO on backend | Built into your Express app |
| Queue | In-process cron (node-cron) | No Redis needed for MVP |

**What changes from Supabase setup:**
- No Supabase client in frontend
- No RLS policies (you enforce tenant isolation in every backend query)
- Auth is your own JWT (not Supabase Auth)
- Realtime is Socket.IO (not Supabase Realtime)
- Direct `pg` / `drizzle-orm` / `prisma` connection to Render PostgreSQL

## Render Free PostgreSQL — Real Facts

On the free tier, your database will expire after 90 days, and you can only have one PostgreSQL instance at a time.

Before the end of 90 days, you will need to back up your databases, delete the PostgreSQL instance from Render, create a new PostgreSQL instance, and populate it from the database backups.

**For PropAgent production: upgrade to Render PostgreSQL Basic-1gb ($7/month) after MVP.**
Free tier is fine for development and first 1–2 clients.

Render Postgres uses the default PostgreSQL port 5432. To use the internal URL, your connecting service and your database must belong to the same account and region. Wherever possible, connect to your database using its internal URL.

Internal URL format:
```
postgresql://USER:PASSWORD@INTERNAL_HOST:5432/DATABASE
```

If your psql connection fails, add `?sslmode=require` at the end of your connection string.

## STEP 1 — Create Render Account + PostgreSQL

1. Go to **render.com** → Sign up with GitHub
2. Dashboard → **New +** → **PostgreSQL**
3. Fill in:
   - Name: `propagent-db`
   - Database: `propagent` (leave blank → auto-generated)
   - User: leave blank → auto-generated
   - Region: **Singapore** (closest to India + Gulf)
   - Plan: **Free**
4. Click **Create Database**
5. Wait ~2 minutes for provisioning

Once created, go to the database Info page:
- Copy **Internal Database URL** (use this for backend connection)
- Copy **External Database URL** (use this for local development + migrations)
- Copy **PSQL Command** (use this to run migrations)

Internal URL format:
```
postgresql://propagent_user:AUTOGENERATED_PASSWORD@dpg-XXXX-a/propagent
```

### `src/db/index.ts`

```typescript
import { Pool } from 'pg';

export const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
  ssl: process.env.NODE_ENV === 'production'
    ? { rejectUnauthorized: false }
    : false,
  max: 10,                    // max connections in pool
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});

// Test connection on startup
pool.connect((err, client, release) => {
  if (err) {
    console.error('DB connection failed:', err.message);
    return;
  }
  client.query('SELECT NOW()', (err, result) => {
    release();
    if (err) {
      console.error('DB query failed:', err.message);
      return;
    }
    console.log('DB connected:', result.rows[0].now);
  });
});
```

```sql
-- ═══════════════════════════════════════════════════════
-- UPDATED_AT AUTO-TRIGGER
-- ═══════════════════════════════════════════════════════
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER tenants_updated_at
  BEFORE UPDATE ON tenants
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

CREATE TRIGGER broker_settings_updated_at
  BEFORE UPDATE ON broker_settings
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

CREATE TRIGGER properties_updated_at
  BEFORE UPDATE ON properties
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();

-- ═══════════════════════════════════════════════════════
-- HELPER VIEWS
-- ═══════════════════════════════════════════════════════

-- Active conversations with lead count
CREATE OR REPLACE VIEW tenant_stats AS
SELECT
  t.id AS tenant_id,
  t.business_name,
  t.plan,
  COUNT(DISTINCT c.id) FILTER (WHERE c.created_at > NOW() - INTERVAL '30 days') AS leads_this_month,
  COUNT(DISTINCT m.id) FILTER (WHERE m.status = 'confirmed') AS meetings_confirmed,
  COALESCE(cp.ai_messages_used, 0) AS ai_used,
  COALESCE(cp.ai_message_limit, 0) AS ai_limit
FROM tenants t
LEFT JOIN conversations c ON c.tenant_id = t.id
LEFT JOIN meetings m ON m.tenant_id = t.id
LEFT JOIN client_plans cp ON cp.tenant_id = t.id
GROUP BY t.id, t.business_name, t.plan, cp.ai_messages_used, cp.ai_message_limit;
```

## STEP 6 — Auth System (Custom JWT — No Supabase)

Since you're not using Supabase Auth, build your own. Simple and clean.

### `src/services/auth.service.ts`

```typescript
import bcrypt from 'bcrypt';
import jwt from 'jsonwebtoken';
import { pool } from '../db';

const SALT_ROUNDS = 12;

export async function hashPassword(plain: string): Promise<string> {
  return bcrypt.hash(plain, SALT_ROUNDS);
}

export async function verifyPassword(plain: string, hash: string): Promise<boolean> {
  return bcrypt.compare(plain, hash);
}

export function signAccessToken(tenantId: string, plan: string): string {
  return jwt.sign(
    { tenant_id: tenantId, plan },
    process.env.JWT_SECRET!,
    { expiresIn: process.env.JWT_EXPIRY || '7d' }
  );
}

export function signRefreshToken(tenantId: string): string {
  return jwt.sign(
    { tenant_id: tenantId },
    process.env.REFRESH_TOKEN_SECRET!,
    { expiresIn: process.env.REFRESH_TOKEN_EXPIRY || '30d' }
  );
}

export function verifyAccessToken(token: string): { tenant_id: string; plan: string } {
  return jwt.verify(token, process.env.JWT_SECRET!) as any;
}
```

### `src/middleware/auth.ts`

```typescript
import { Request, Response, NextFunction } from 'express';
import { verifyAccessToken } from '../services/auth.service';

export interface AuthRequest extends Request {
  tenantId?: string;
  plan?: string;
}

export function requireAuth(req: AuthRequest, res: Response, next: NextFunction) {
  const header = req.headers.authorization;
  if (!header || !header.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'No token' });
  }
  try {
    const payload = verifyAccessToken(header.replace('Bearer ', ''));
    req.tenantId = payload.tenant_id;
    req.plan = payload.plan;
    next();
  } catch {
    return res.status(401).json({ error: 'Invalid token' });
  }
}
```

### `src/realtime/index.ts`

```typescript
import { Server as SocketServer } from 'socket.io';
import { Server } from 'http';
import { verifyAccessToken } from '../services/auth.service';

export function initSocketIO(httpServer: Server) {
  const io = new SocketServer(httpServer, {
    cors: { origin: process.env.FRONTEND_URL, methods: ['GET', 'POST'] }
  });

  io.use((socket, next) => {
    const token = socket.handshake.auth.token;
    try {
      const payload = verifyAccessToken(token);
      socket.data.tenantId = payload.tenant_id;
      next();
    } catch {
      next(new Error('Unauthorised'));
    }
  });

  io.on('connection', (socket) => {
    const tenantId = socket.data.tenantId;
    // Each broker joins their own room — only sees their own events
    socket.join(`tenant:${tenantId}`);

    socket.on('disconnect', () => {
      socket.leave(`tenant:${tenantId}`);
    });
  });

  return io;
}
```

## STEP 9 — 90-Day Free DB Rotation (Critical)

Before the end of 90 days, you will need to back up your databases, delete the PostgreSQL instance from Render, create a new PostgreSQL instance, and populate it from the database backups.

**Automate the backup so you don't lose data:**

```bash
# Run this locally every 2 weeks (or set up as a cron on your machine)
# Replace with your External DB URL from Render dashboard

pg_dump "postgresql://USER:PASS@host.render.com:5432/propagent?sslmode=require" \
  --format=custom \
  --no-acl \
  --no-owner \
  -f backup-$(date +%Y%m%d).dump

# To restore to a new Render DB:
pg_restore \
  --no-acl \
  --no-owner \
  -d "postgresql://USER:PASS@new-host.render.com:5432/propagent?sslmode=require" \
  backup-20250601.dump
```

**Set a calendar reminder: every 80 days → run backup + rotate DB.**
